# Security Policy

## Supported Versions

The following versions of this project are currently receiving security updates:

| Version         | Supported          |
| --------------- | ------------------ |
| 0.1.0-alpha     | :white_check_mark: |

All versions from `0.1.0-alpha` onward will continue to be supported with security updates until this policy is updated.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it by simon.roy1211@gmail.com.

When reporting, please include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- The version affected
- Any potential impact you've identified

**What to expect:**
- You'll receive an acknowledgment within a few days of your report.
- We'll provide an initial assessment and status update in the following days.
- If the vulnerability is **accepted**, we'll work on a fix and coordinate a disclosure timeline with you. You'll be credited in the release notes unless you prefer to remain anonymous.
- If the vulnerability is **declined**, we'll explain our reasoning and, where possible, suggest alternatives or workarounds.

Please do **not** publicly disclose the vulnerability until we've had a chance to address it.
